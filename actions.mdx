## What are Actions?

The **Actions** page has two main purposes:

1. **API Action Templates** — pre-configured tracker API endpoint definitions that power the [Tracker Action steps](/automations/steps/tracker-actions) in the automation builder.
2. **Built-in Composite Actions** — purpose-built tools for common operations that go beyond a single API call, like the **Bulk Offer Upload** for Affise.

---

## Bulk Offer Upload (Affise)

The Bulk Offer Upload tool lets you create multiple Affise offers at once from a single text or CSV input — no manual clicking through the Affise UI.

### How it works

1. **Select your Affise connection** from the connection dropdown (admin connection required).
2. **Load a model offer** — enter an existing offer ID to use as a template. The tool fetches all of its settings and displays them in collapsible sections (General, Targeting, Tracking, Payouts, Caps, etc.).
3. **Paste your offer data** — paste freeform text or a CSV table describing your new offers. The AI parser extracts offer rows and maps fields automatically (title, URL, preview URL, and any other overrides you describe).
4. **Review the parsed table** — each offer row is editable before creation. You can also set global overrides that apply to all offers.
5. **Configure output options** — optionally enable affiliates by PID and generate tracking links. Choose which data to include in the results (preview URL, payout, caps).
6. **Create offers** — the tool creates each offer sequentially, copies payout rules from the model, enables the specified affiliates, and generates tracking links.

### Results

For each created offer you get:

| Field | Description |
|---|---|
| **Tracking URL** | Per-affiliate tracking link (generated via API, or constructed from the tracking domain as a fallback) |
| **Preview** | The offer preview/landing page URL |
| **Payout** | Affiliate payout per goal (e.g. `$3.50 (Goal: Default)`) |
| **Caps** | Cap rules summary (e.g. `200/day · Conversions`) |

### What gets copied from the model offer

- All scalar settings (title, URL, redirect type, privacy, tracking domain, etc.)
- Payout rules (general payments — personal/partner payouts are not transferred)
- Edit-only settings applied in a follow-up call: `is_cpi`, `hide_payments`, `allow_impressions`, `enabled_commission_tiers`, etc.

### What must be configured manually in Affise after creation

- **Cap rules** — the Affise API does not accept cap rules on offer creation. Set them in Affise under the offer's Caps tab.
- **Partner / personal payouts** — these are affiliate-specific and cannot be transferred via the API.
- **Commission tiers** — applied via edit call but may need manual review in complex setups.

<Note>
  Offers are created sequentially (one at a time) to avoid rate limiting. For large batches, the request may take a minute or more.
</Note>

## API Action Templates

Action templates power the [Tracker Action Get, Edit, and Add steps](/automations/steps/tracker-actions) in the automation builder. Each template defines:

- Which tracker connection it belongs to
- The API endpoint path and HTTP method (GET, POST, PUT, PATCH)
- The parameters or request body it accepts

You configure templates once and reuse them across multiple automations.

---

## Who Should Use Action Templates?

Templates are for users who want to go beyond standard stats-pulling and interact with their tracker's API directly — reading specific records, updating settings, or creating new entries.

If you just need to pull stats and send emails, you don't need to configure any action templates.

---

## Creating an Action Template

1. Go to **Actions** in the left sidebar
2. Click **Add Action Template**
3. Select the **connection** (tracker) this action will use
4. Enter the **endpoint path** (e.g. `/3.0/affiliate/{affiliate_id}` for Affise)
5. Select the **HTTP method** (GET, POST, PUT, PATCH, DELETE)
6. Define any **parameters** the endpoint accepts and their types
7. Optionally provide a **test request** to verify the endpoint works
8. Save the template

Once saved, the template appears as a selectable option inside Tracker Action steps in the automation builder.

## Action Types and Use Cases

### GET Actions (Tracker Action Get)

Used to read data not available through standard stat types. Examples:

- Get a specific affiliate's details by ID
- Fetch the current settings for a specific offer
- Read conversion records for approval

### POST / PUT Actions (Tracker Action Edit, Tracker Action Add)

Used to write data back to your tracker. Examples:

- Update an affiliate's payout cap
- Change an offer's status to active or paused
- Create a new postback URL for an affiliate

---

## Using Tracker API Documentation

To create action templates, you need to know your tracker's API endpoints. Consult your platform's API documentation:

- **Affise**: Available in your Affise admin panel under Settings → API Documentation, or at your Affise account manager
- **Binom**: Available in Binom's own settings panel under API
- **CAKE**: Available in CAKE's affiliate portal documentation

<Tip>
  Start with read-only GET actions before building any write actions (Edit or Add). This lets you verify the endpoint and response format without risk of accidentally modifying data.
</Tip>

---

## Testing an Action Template

After creating a template, use the **Test** button to make a real API call. The response is displayed so you can verify the endpoint returns what you expect and understand the field structure before building an automation around it.
